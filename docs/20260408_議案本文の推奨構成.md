# 議案本文の推奨構成

## 目的

石垣市議会版みらい議会において、議案詳細ページの本文を市民にとって読みやすく、かつ継続運用しやすい形で整備するための推奨構成を整理する。

本メモは、既存の `bill_contents` テーブルおよび Markdown 表示の仕組みを前提とし、現時点で最も実装負荷が低く、運用しやすい方法を示す。

## 前提

- 議案本文は `bill_contents.content` に Markdown 形式で保存される
- 議案詳細ページでは Markdown の見出し構造をそのまま表示する
- 難易度別コンテンツは `bill_contents.difficulty_level` で切り替える
- 現在の表示対象は主に `normal` と `hard`

## 推奨方針

現時点では、専用カラムや JSON 構造を追加するのではなく、`bill_contents.content` の Markdown 見出しとして情報を持つ方式を推奨する。

理由:

- 既存実装をそのまま活かせる
- Admin 側の入力方式を大きく変えずに運用できる
- 本文の編集自由度を保てる
- まず少数の注目議案から改善しやすい

## 推奨セクション構成

`normal` および `hard` の本文では、可能な限り以下の順に見出しを置く。

1. この議案のポイント
2. この議案が必要な理由
3. 主な論点
4. 影響を受ける人
5. 関連リンク

## 項目名について

### 1. この議案のポイント

- 最初に「何を決める議案か」がわかるようにする
- 箇条書きまたは短い段落で記述する

### 2. この議案が必要な理由

- なぜこの議案が提出されたのかを説明する
- 背景、課題、現状の問題点を書く

### 3. 主な論点

- 議会で議論になりやすい点を整理する
- 「意見が分かれるところ」よりも中立的な表現として使用する

### 4. 影響を受ける人

- 市民の誰に関係するのかを具体的に示す
- 例:
  - 子育て世帯
  - 高齢者
  - 市内事業者
  - 漁業関係者

### 5. 関連リンク

- 公式資料や参考ページがある場合に記載する
- リンクが無い場合は省略してよい

## `normal` と `hard` の使い分け

### `normal`

- 市民向けの標準版
- 最初の1文で議案の意味が伝わるようにする
- 専門用語は少なめにし、必要なら言い換える
- 1文を短めにする

### `hard`

- 詳細版
- 制度的背景、法的背景、実務上の論点まで含める
- 専門用語の使用を許容する
- ただし、見出し構成自体は `normal` とそろえることが望ましい

## 推奨 Markdown 例

```md
# ○○議案とは

## この議案のポイント

- 何を決める議案か
- 何が変わるのか

## この議案が必要な理由

いま何が課題になっているのか、この議案がなぜ必要なのかを説明する。

## 主な論点

- 財源
- 対象範囲
- 実施方法

## 影響を受ける人

- ○○する市民
- ○○業の事業者

## 関連リンク

- [石垣市議会](https://www.city.ishigaki.okinawa.jp/soshiki/gikai/index.html)
```

## 将来の拡張について

将来的に以下の要件が強くなった場合は、本文 Markdown 運用から専用構造への移行を検討する。

- 各項目を UI 上でカード表示したい
- 項目ごとに出し分けたい
- Admin でフォーム入力したい
- 関連リンクを複数件管理したい

その場合の候補:

1. `bill_contents` に専用カラムを追加する
2. `structured_content` のような JSON カラムを追加する

ただし現時点では、Markdown 見出しで運用する方式を第一推奨とする。
