## 今後の運用ルール

### 1. 普段の試作

- 議案の説明文作成
- タグ案の検討
- 表示確認

これらはまず開発用 DB で行う。

### 2. 公開したい変更

- トップに出す議案
- `bill_contents`
- `is_featured`
- タグ
- 会期データ

これらは、本番反映用 SQL を作ってから本番 DB に入れる。

### 3. 本番反映の基本手順

1. 開発用 DB で内容を確認する
2. 本番用 SQL を用意する
3. 本番 Supabase の SQL Editor で実行する
4. 必要なら `/api/revalidate` でキャッシュを更新する
5. 公開サイトで表示確認する
