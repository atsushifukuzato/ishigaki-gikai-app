# 既存議案の整備優先順

## 前提

- 対象は石垣市議会版の実議案のみとする
- `packages/seed` に入っているサンプル議案は整備対象に含めない
- 公開サイト `web` に出る議案から優先して整える

## 優先順位

既存議案は、まず以下の順で整備する。

1. トップに出ている注目議案
2. `published` の実議案
3. `coming_soon` の実議案
4. `draft` の実議案

公開中の議案を先に整えることで、利用者が実際に見るページから順に品質を上げる。

## 直近の対象

### 1. 議員提出議案第2号 尖閣諸島周辺海域における漁船用係留ブイの設置を求める意見書
- 石垣市の漁業、安全保障、離島政策にまたがるテーマで、石垣市議会版らしさが強い
- `normal` / `hard` の両方で、背景と論点を整理する価値が高い

### 2. 議案第13号 令和8年度石垣市一般会計予算
- 市民生活との接点が広く、トップ掲載議案として重要度が高い
- 「何を優先する予算か」「財源に無理がないか」を見出しで整理しやすい

### 3. 議案第21号 石垣市犯罪被害者等支援条例
- 支援対象や制度の実効性など、市民目線で整理すると理解しやすくなる
- `normal` はやさしく、`hard` は制度面まで踏み込む差分をつけやすい

## 各議案で入れる推奨構成

`bill_contents.content` では、以下の見出し構成を基本とする。

1. この議案のポイント
2. この議案が必要な理由
3. 主な論点
4. 影響を受ける人
5. 関連リンク

## 作業手順

1. 対象が石垣市議会の実議案かを確認する
2. 既存の `normal` 内容を推奨構成に整理する
3. `hard` も同じ見出し構成で、制度や実務の背景を補う
4. 本番反映用 SQL を作る
5. 本番 DB へ反映後、公開ページで表示確認する

## 補足

- `normal` は市民向けの標準版として、最初の数行で議案の意味がつかめる表現を優先する
- `hard` は制度、財政、運用上の論点まで含めた詳細版とする
- `関連リンク` は、石垣市議会、石垣市、国の関係機関など信頼できる一次情報を優先する
